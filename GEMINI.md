# Mini OS プロジェクト概要

32ビット x86 (IA-32) アーキテクチャをターゲットとした、学習用マルチスレッドオペレーティングシステム。12日間でOSの基礎からプリエンプティブマルチタスクまでを段階的に実装するカリキュラム形式。

## 主な機能と技術スタック
- **アーキテクチャ**: 32-bit x86 (プロテクトモード)
- **言語**: C (Freestanding), x86 Assembly (NASM)
- **ビルドツール**: i686-elf-gcc, nasm, GNU Make
- **エミュレータ**: qemu-system-i386
- **主要機能**:
  - ブートローダ (リアルモードからプロテクトモードへの移行)
  - VGA テキストモード描画
  - シリアル通信によるデバッグ出力
  - IDT/PIC による割り込み管理 (タイマー、キーボード)
  - ラウンドロビン方式のプリエンプティブスケジューリング
  - TCB (Thread Control Block) によるコンテキストスイッチ
  - PS/2 キーボードドライバ (リングバッファ、スキャンコード変換)

## ビルドと実行

### 主要なコマンド (day12_completed または各日ディレクトリにて)
- `make all`: OSイメージ (`os.img`) のビルド
- `make run`: QEMU で OS を起動 (GUIあり、シリアルデバッグ出力あり)
- `make run-nogui`: GUIなしで QEMU を起動 (シリアル出力のみ)
- `make clean`: 生成されたバイナリ等の削除
- `make check-env`: 開発ツールのインストール確認
- `make analyze`: cppcheck および gcc による静的解析
- `make quality`: コードの複雑度やマジックナンバーのチェック

### テストの実行
`make test` コマンドで単体テストおよびコンパイルテストを実行可能。出力が膨大になるため、必ずリダイレクトすること。
```bash
make test > makeout.txt 2>&1
```

## 開発規約と構造

### ディレクトリ構造 (day12_completed 例)
- `src/`: ソースコード
  - `boot/`: ブートローダ、割り込みスタブ、エントリポイント
  - `kernel.c`: スケジューラおよびカーネルの中核
  - `keyboard.c`: キーボードドライバ
- `include/`: ヘッダーファイル
- `linker/`: リンカースクリプト (`kernel.ld`)
- `tests/`: テストコードとモックハードウェア
- `docs/`: 設計ドキュメント (`ARCHITECTURE_ja.md` 等)

### コーディングスタイル
- **言語**: C (gnu99), NASM (elf32/bin)
- **インデント**: 4スペース (タブ禁止)
- **命名規則**: 関数・変数は `snake_case`、マクロ・定数は `UPPER_SNAKE_CASE`
- **設計方針**:
  - ヘッダーで API を公開し、実装は `src/` に配置。
  - マジックナンバーを避け、ヘッダーで定数 (PIC/PIT/IDT/VGA等) を管理。
  - 単一責任原則に基づき、関数を小さく保つ (32行以内を推奨)。
- `clang-format` を使用 (`make format`)。

### メモリレイアウト
- `0x7C00`: ブートセクタ
- `0x100000` (1MB): カーネル実行アドレス
- `0x200000` (2MB): カーネルスタック
- `0xB8000`: VGA テキストバッファ

## 注意事項
- **回答言語**: 日本語を使用すること。
- **トーン**: 簡潔に回答し、過度な礼儀表現は不要。
- **リダイレクト**: `make test` 等の実行時は `makeout.txt` へ出力をリダイレクトすること。
